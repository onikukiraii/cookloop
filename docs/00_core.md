# 冷蔵庫管理アプリ 体験ビジョン

## コンセプト

> 冷蔵庫を開けなくても、今夜の献立が決まる

食材を買って、使って、また買う。このループの中で起きる「気づいたら賞味期限が切れていた」「何を買えばいいかわからない」「今夜何作ろう」という摩擦をすべて取り除く。

在庫管理が目的ではなく、**冷蔵庫の中身と今夜の夕食が繋がること**がゴール。

---

## 原体験

- 野菜や肉を買ったのに、使わないまま賞味期限を迎えてしまう
- 冷蔵庫の隅に特定の食材だけ残り続ける
- スーパーで「あれ、家に何があったっけ」となる
- 毎日「今夜何作ろう」を一から考えるのが面倒

---

## 食材のライフサイクル

このアプリが管理するのは、食材の一生全体。

```
買い物リストに追加
  （手動 / レシピ提案から / 定番フラグで自動）
      ↓
スーパーでチェック
      ↓
冷蔵庫に自動登録
  （賞味期限はキャッシュから即推定・残量は「たっぷり」）
      ↓
日々使う
  （残量を手動で更新：たっぷり → 半分 → 少し）
      ↓
残量「少し」
  （定番フラグありなら買い物リストへ自動追加）
      ↓
使い切り・削除
```

---

## 3つの主要体験

### 1. 買い物する

**買い物リストは3つの経路で育つ**

- 在庫を見て「そろそろ買おう」と手動追加
- 「この料理作りたい」→ 足りない食材がリストに自動追加
- 定番フラグの食材が残量「少し」になったら自動追加

**リストは2つのセクションに分かれる**

```
【買うもの】
☐ 豚バラ肉        ← 手動 or レシピから追加
☐ 玉ねぎ

【そろそろなくなりそう】
☐ 納豆            ← 定番フラグで自動追加
☐ 卵
```

**スーパーでチェックしたら即冷蔵庫へ**

チェックした瞬間に冷蔵庫へ自動登録される。
賞味期限は食材マスタのキャッシュから即推定。入力不要。

---

### 2. 冷蔵庫を把握する

**期限が近い食材が目立つ**

今日・明日・2日以内の食材が視覚的に危険度を示す。
冷蔵庫を開けなくても「何をいつまでに使わないといけないか」が一目でわかる。

**残量は3段階で管理**

| ステータス | 意味 |
|---|---|
| たっぷり | メイン食材として使える |
| 半分 | まだある |
| 少し | 使い切りに近い、定番なら買い物リストへ |

グラム数は管理しない。「少しのほうれん草」という粒度がわかれば十分。

**調味料は別で管理**

賞味期限は管理しない。残量と「切らしたくない」フラグだけ。

---

### 3. 今夜の献立を決める

**AIが冷蔵庫全体を見て提案する**

単に「期限が近い順」ではなく、**冷蔵庫全体を満遍なく消費できる料理**を提案する。
特定の食材だけ毎週残り続ける状態を避けることが最優先。

**提案の2パターン**

- 「今夜何作ろう」→ 冷蔵庫全体のバランスを見て3候補
- 「この食材を使いたい」→ 指定食材を必ず含む料理を提案

**ホットクックにも対応**

ホットクックで作れる料理には、メニュー番号を添える。
ただし公式レシピに縛られない。冷蔵庫にある食材で代替・省略できる場合はAIが判断する。

```
例：肉じゃがを提案
  → こんにゃくがなければ省略
  → 三つ葉がなくても成立
  → ホットクック No.0156 で作れます
```

**手動調理も同列で提案**

ホットクックにない料理も提案する。手順はAIがその場で生成。

**提案には手順も表示する**

- ホットクックレシピ → DBに保存済みの公式手順を表示
- 手動調理 → AIがその場で生成した手順を表示

---

## 食材検索の名前揺れ対応

「人参」と入力しても「にんじん」がヒットする。

OpenSearchとkuromojiアナライザーで表記揺れを吸収する。
別名（aliases）はGeminiが自動生成するため、ユーザーが手動で登録する必要はない。

```
「人参」と入力
  → OpenSearchが「にんじん」にマッチ
  → 在庫として登録
```

誤った別名が登録された場合は設定画面から修正できる。

---

## 食材マスタが育つほど楽になる

初回登録時だけGeminiで賞味期限を推定する。
一度登録した食材はマスタにキャッシュされ、次回からは即登録。

ホットクック654レシピのスクレイピング時にマスタを事前投入するため、
一般的な食材はアプリ初日からキャッシュ済みの状態で使い始められる。

---

## 定番フラグ

毎回必ず家にあってほしい食材に立てるフラグ。

- 残量が「少し」になった時点で買い物リストに自動追加
- 切れてから買うのではなく、なくなりそうなときに買える
- 納豆・卵・牛乳などが典型

---

## このアプリが解決しないこと

- 食材の正確なグラム数管理（残量の3段階で十分と判断）
- 複数人での共有・家族間の同期（個人利用前提）
- 栄養バランスの管理（献立提案の主軸は食材消費）
